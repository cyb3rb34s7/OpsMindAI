# Onboarding Agent

## Purpose
Transform repositories into operational understanding.

## Why This Matters
Other agents depend on onboarding-generated context.

## Intelligence Model
Deterministic heuristics first.
LLM reasoning second.

## Output Artifacts
- project_index.md
- service_map.md
- tech_stack.md
- open_questions.md

## Large Repository Strategy
Repositories above threshold are partially scanned to prevent
recursive runtime explosion.

## Runtime Guarantees
- explicit warnings
- visible partial failures
- persisted artifacts
- operational traceability
